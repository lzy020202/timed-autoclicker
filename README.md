# 定时连点器

一个仅支持 Windows 10/11 的简单鼠标左键连点工具。

## 功能

- 指定精确到毫秒的开始时间
- 设置相对开始时间提前或延后若干毫秒
- 手动保存最近 20 个开始时间，可从下拉列表直接选择
- 设置点击间隔与点击次数（`0` 为持续点击）
- 使用系统时间、北京时间、淘宝服务器时间或饿了么网页响应时间计算时间偏移
- 全局 `F8` 开始/紧急停止
- 按 `F7` 记录目标位置，到点后自动移动鼠标并点击
- 不修改 Windows 系统时钟

## 打包 EXE

1. 在 Windows 安装 Python 3.11 或更高版本，并勾选 **Add Python to PATH**。
2. 双击 `build.bat`。
3. 成品位于 `dist\定时连点器.exe`。

也可以直接运行源码：

```bat
py autoclicker.py
```

## 在 GitHub 云端打包

项目内置了 `.github/workflows/build-windows.yml`：上传到 GitHub 后，会使用免费的 Windows 构建环境生成 EXE。

1. 在 GitHub 新建一个仓库。
2. 将本目录全部文件（包括隐藏的 `.github` 文件夹）上传到仓库。
3. 打开仓库的 **Actions** 页面，选择 **Build Windows EXE**。
4. 点击 **Run workflow**，再确认运行。
5. 构建完成后打开该次运行，在页面底部的 **Artifacts** 下载 `TimedAutoClicker-Windows`。
6. 解压下载的 ZIP，即可得到 `TimedAutoClicker.exe`。

打包只会在 Actions 页面手动点击 **Run workflow** 后运行，普通代码提交不会自动打包。

## 使用说明

1. 选择时间源并点击“立即校时”。系统时间无需联网；平台时间需要联网。
2. 输入开始时间后点击“保存并采用”；不填写毫秒时会自动补成 `.000`。只有保存后的时间才会生效。
3. 也可以从“时间记录”下拉框直接选择已保存的时间。
4. 如需调整，选择“提前”或“延后”并填写毫秒数。
5. 把鼠标移到目标位置并按 `F7`；也可点击“3 秒后记录”，再移动鼠标。
6. 点击“开始 / 等待”。到点后鼠标会自动移动到记录位置并开始点击。
7. 随时按 `F8` 停止。

## 精度说明

- 淘宝时间通过公开时间戳接口取得，程序用请求往返时间的中点估算网络延迟。
- 北京时间通过阿里云公共 NTP 服务器同步，按 UTC+8 显示，并使用 NTP 四时间戳算法估算网络延迟。
- 饿了么没有面向普通客户端的公开高精度校时接口，本工具使用其网页 HTTP `Date` 响应头，因此通常只有约 1 秒粒度；界面会显示测得的偏差。
- Windows 调度、网络抖动和目标应用响应都会影响实际点击时刻，本工具不保证抢购或下单成功。
